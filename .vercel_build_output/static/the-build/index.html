I've got a boiler plate project for you here, continue reading if you want to know a little more about how it works!

Before we start, it's important to note most the tools chosen for this project are still in beta (SvelteKit & TailwindCSS JIT) as of the time of writing this (23rd June 2021).

These tools were picked for fun, I was mostly just interested in checking out a few things:

  • How these tools are coming along.
  • Dev experience using a CSR tool in an SSR app.
  • Any excuse to try out three.js!

1

Firstly, I grabbed some assets off the web and threw this together in Blender. I didn't really care about the scene too much, my priority was making the website - I was a little out of my depth here!

Once I was happy with it - I exported the scene as a .gltf file format.

2

Now the fun part - It's Web Dev time.

I created the SvelteKit App with Tailwind JIT following this great post. From there I installed three.js with it's type definitions.

Since I want to load a .gltf scene, lets set up GLTFLoader from three.js. This turned out to be a little tricky, but I got it working by grabbing the code from the GitHub, and I added it to the src directory of my project as GLTFLoader.ts. I then changed the import to be from 'three'. This was a work around for this error. Hopefully this gets fixed soon! Now simply import your loader, don't forget to destructure it!

import { GLTFLoader } from '../GLTFLoader'

3

Finally, it's time to actually put our scene on the page. To do this we have to remember that three.js relies on running client side as it references a HTMLCanvasElement in the DOM you want three to render it's cool 3D scene to.

Nothing a simple onMount() can't solve! Then for loading your scene, I followed this nice post.

The rest is nothing more than fancy Svelte / TailwindCSS animations - Happy hacking!
